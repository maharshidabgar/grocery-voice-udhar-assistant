# 🤝 Contributing to Grocery Voice Udhar Assistant

Thank you for your interest in contributing to this project!

We welcome bug reports, feature requests, documentation improvements, and code contributions.

---

# 📋 Before You Start

Please make sure you:

- Read the project documentation.
- Check existing Issues before creating a new one.
- Discuss major feature requests before starting implementation.

---

# 🚀 Getting Started

## 1. Fork the Repository

Create your own copy of the repository on GitHub.

## 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/Grocery-Voice-Udhar-Assistant.git
```

## 3. Create a Virtual Environment

```bash
python -m venv .venv
```

Activate it (Windows):

```bash
.venv\Scripts\activate
```

## 4. Install Dependencies

```bash
pip install -r requirements.txt
```

---

# 🌿 Create a New Branch

Use a descriptive branch name.

Examples:

```text
feature/pdf-export
feature/voice-improvements
feature/dashboard
bugfix/report-export
```

---

# 💻 Coding Guidelines

Please follow these guidelines:

- Write clean and readable Python code.
- Use meaningful variable and function names.
- Keep functions focused on a single responsibility.
- Add comments only where necessary.
- Follow the existing project structure.

---

# ✅ Before Submitting

Please ensure:

- The project runs without errors.
- New features are tested.
- Existing functionality is not broken.
- Code is properly formatted.

---

# 📤 Commit Changes

Example:

```bash
git add .
git commit -m "Add PDF export improvements"
```

Push your branch:

```bash
git push origin feature/pdf-export
```

---

# 🔀 Open a Pull Request

When opening a Pull Request:

- Clearly explain the changes.
- Reference related Issues if applicable.
- Include screenshots for GUI changes.

---

# 🐞 Reporting Bugs

Please include:

- Operating System
- Python Version
- Error Message
- Steps to Reproduce
- Expected Behavior

# 💡 Feature Requests

When suggesting a feature, please describe:
